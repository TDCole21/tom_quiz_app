Data Protection

  1. Passwords:

    2. All passwords are hashed before being stored on a publically hosted MySQL database on Microsoft Azure.
    For the purpose of this document, a password that has been hashed is called a secure password and an unsecure password if not hashed.
    Not even the web application developers are able to know secure passwords.
    If the User forgets their password and requires it to be reset, they are currently unable to do this without the web application developer.
    The user will need to contact the developer who will create a tempaoray unsecure password, which the User can then use to login and creat a new secure password.

  2. Emails:

    3. User's email addresses will not be made public on the web application and are for contacting the user about information about the web application.
    However, these features are not yet implemented in the web application, but the data is being collected in preparation for the update.


Data Use

  1. Personal Information:

    2. User information is not sold, and only used for the personalisation of the web application.
    By creating an account you are giving permission for the web application to use display your "User Name".

  2. Account deletion:

    3. When you delete your account, all associated information with you account is also deleted.
